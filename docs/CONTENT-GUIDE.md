# Руководство по наполнению базы знаний

## Формат данных

База знаний хранится в файле `src/data/knowledge-base.js`. Файл содержит два массива: `CATEGORIES` (категории) и `ARTICLES` (статьи).

## Как добавить статью

1. Открой файл `src/data/knowledge-base.js`
2. Найди массив `ARTICLES`
3. Скопируй любой существующий блок статьи
4. Измени поля:

```js
{
  id: 15,                    // Уникальный номер (следующий по порядку)
  category: "work",          // ID категории (см. таблицу ниже)
  title: "Название статьи",
  summary: "Краткое описание для карточки (1 строка)",
  content: `Полный текст статьи.

Переносы строк создаются через пустую строку между абзацами.

• Списки делаются через символ •
• Второй пункт`
}
```

5. Сохрани файл, закоммить и запуши

## Доступные категории

| ID         | Название            |
| ---------- | ------------------- |
| `newbie`   | Новому сотруднику   |
| `company`  | О компании          |
| `work`     | Организация работы  |
| `life`     | Корпоративная жизнь |
| `feedback` | Обратная связь      |

## Как добавить категорию

Добавь объект в массив `CATEGORIES`:

```js
{ id: "safety", name: "Безопасность", icon: "🔒", color: "#6366f1", desc: "Правила информационной безопасности" }
```

Затем используй `id` новой категории в статьях.

## Внутренние ссылки

Чтобы создать ссылку на другую статью внутри текста, используй синтаксис:

```
[[ID_СТАТЬИ|Текст ссылки]]
```

Пример:

```
Подробнее — в карточке «[[10|Сервисы для работы]]»
```

Это создаст кликабельную ссылку на статью с id=10.

## Правила оформления текста

- Абзацы разделяй пустой строкой (`\n\n`)
- Для списков используй `•` или `→`
- Emoji в начале разделов для визуального разделения (🌴, ⚠️, 📌 и т.д.)
- Заголовки разделов внутри статьи — просто жирная строка (без markdown-синтаксиса)
- Не используй HTML-теги

## Чеклист перед публикацией

- [ ] Уникальный `id` (не совпадает с существующими)
- [ ] Правильный `category` (существует в CATEGORIES)
- [ ] `summary` — 1 строка, до 60 символов
- [ ] `content` — вычитан, нет опечаток
- [ ] Внутренние ссылки `[[id|text]]` ведут на существующие статьи
- [ ] Файл собирается без ошибок (`npm run build`)
