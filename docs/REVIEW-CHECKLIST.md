# Чеклист ревью: kn-base

Используй этот чеклист при ревью Pull Request.

## Общее

- [ ] PR имеет понятное описание и ссылку на Task ID из [BACKLOG.md](BACKLOG.md)
- [ ] Ветка названа по конвенции: `feat/`, `fix/`, `test/`, `docs/`, `content/`
- [ ] CI проходит (lint + test + build)
- [ ] Нет console.log / debugger в коде
- [ ] Нет закомментированного кода

## Код

- [ ] Компоненты используют CSS-переменные из `global.css` (не хардкод цветов)
- [ ] Шрифты: `var(--font-display)` для заголовков, `var(--font-body)` для текста
- [ ] Новые компоненты имеют единообразную структуру с существующими
- [ ] `useCallback` / `useMemo` для дорогих вычислений и стабильных ссылок
- [ ] Нет утечек состояния (эффекты с cleanup где нужно)

## Данные

- [ ] Новые статьи имеют уникальный `id`
- [ ] `category` ссылается на существующую категорию
- [ ] `summary` — краткий (до 60 символов)
- [ ] Внутренние ссылки `[[id|text]]` валидны
- [ ] Контент вычитан (нет опечаток, актуальная информация)

## UI/UX

- [ ] Проверено на мобильном viewport (375px)
- [ ] Hover-состояния работают
- [ ] Анимации не дёргаются
- [ ] Breadcrumbs корректны на всех уровнях вложенности
- [ ] Scroll-to-top при навигации

## Чат

- [ ] Чат не теряет историю при сворачивании
- [ ] Ошибки API отображаются пользователю
- [ ] API-ключ не логируется и не сохраняется в storage

## Тесты

- [ ] Новый функционал покрыт тестами
- [ ] Существующие тесты не сломаны
- [ ] Тесты не зависят от порядка выполнения
