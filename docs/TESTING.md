# Стратегия тестирования: kn-base

## Пирамида тестов

```
        ┌───────┐
        │  E2E  │  Playwright — 3-5 сценариев
        ├───────┤
        │ Comp. │  React Testing Library — 8-12 тестов
        ├───────┤
        │ Unit  │  Vitest — 15-20 тестов
        └───────┘
```

## 1. Unit-тесты (Vitest)

**Файл:** `src/__tests__/unit/`

### Данные (`knowledge-base.js`)

- Все статьи имеют уникальные id
- Все статьи ссылаются на существующую категорию
- Все категории имеют обязательные поля (id, name, icon, color, desc)
- Все статьи имеют обязательные поля (id, category, title, summary, content)
- Content не пустой ни у одной статьи

### Поиск (`useSearch`)

- Поиск по заголовку находит статью
- Поиск по содержимому находит статью
- Пустой запрос возвращает пустой массив
- Регистронезависимый поиск
- Поиск по частичному совпадению

### Парсер ссылок (`renderContent`)

- Текст без ссылок возвращается как есть
- `[[10|Сервисы]]` рендерит кликабельный span
- Несколько ссылок в одном тексте
- Невалидный id — ссылка всё равно рендерится (graceful degradation)

## 2. Компонентные тесты (React Testing Library)

**Файл:** `src/__tests__/components/`

### CategoryGrid

- Рендерит все 5 категорий
- Каждая плитка показывает название, описание, счётчик статей
- Клик вызывает onSelect с правильным id

### ArticleList

- Рендерит все статьи выбранной категории
- Показывает нумерацию
- Клик на статью вызывает onArticleClick

### ArticleView

- Рендерит заголовок и контент статьи
- Показывает бейдж категории
- Показывает блок «Ещё в разделе» (если есть связанные статьи)
- Не показывает блок «Ещё в разделе» для категории с 1 статьёй

### Breadcrumb

- На главной не рендерится
- В категории: Главная → Категория
- В статье: Главная → Категория → Статья
- Клик на «Главная» вызывает goHome

### SearchBar + SearchResults

- Ввод текста фильтрует статьи
- Результаты показывают бейджи категорий
- «Ничего не найдено» при отсутствии результатов

### ChatWidget

- FAB-кнопка рендерится
- Клик открывает окно чата
- Повторный клик закрывает
- В пустом чате — подсказки-кнопки
- Клик на подсказку заполняет поле ввода

## 3. E2E-тесты (Playwright)

**Файл:** `e2e/`

### Сценарий 1: Навигация по контенту

1. Открыть главную
2. Проверить наличие 5 категорий
3. Кликнуть на «Организация работы»
4. Проверить breadcrumbs: Главная → Организация работы
5. Кликнуть на «Зарплата и выплаты»
6. Проверить breadcrumbs: Главная → Организация работы → Зарплата и выплаты
7. Проверить наличие блока «Ещё в разделе»
8. Кликнуть «Главная» в breadcrumbs → вернуться на главную

### Сценарий 2: Поиск

1. Ввести «отпуск» в поиск
2. Проверить наличие результатов
3. Кликнуть на результат
4. Проверить открытие статьи

### Сценарий 3: Чат-виджет

1. Кликнуть FAB-кнопку
2. Проверить открытие окна чата
3. Проверить наличие подсказок
4. Кликнуть подсказку → текст в поле ввода
5. Кликнуть «‒» → окно закрылось
6. Кликнуть FAB → окно открылось, подсказки на месте

### Сценарий 4: Мобильный viewport (375×812)

1. Открыть главную → 1 колонка
2. Открыть чат → на полную ширину
3. Навигация работает

## 5. CI интеграция

```yaml
# .github/workflows/test.yml
name: Test
on: [push, pull_request]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
      - run: npm ci
      - run: npm run lint
      - run: npm run test
      - run: npm run build
```

E2E-тесты запускаются отдельно перед мержем в main (дорогие по времени).
