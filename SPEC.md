## 1. Описание проекта

Внутренний портал базы знаний для сотрудников компании AdCorp (рекламная сеть с брендами ClickAdilla, MyBid, OnClicka). Портал предоставляет структурированную информацию о компании, рабочих процессах и корпоративной жизни, а также AI-ассистента (чат-бот), отвечающего строго по содержимому базы знаний.

### 1.1 Цели

- Единая точка входа для всей внутренней документации
- Быстрый поиск ответов на рабочие вопросы
- Снижение нагрузки на HR и административные отделы
- Ускорение адаптации новых сотрудников

### 1.2 Целевая аудитория

- Новые сотрудники (онбординг)
- Действующие сотрудники (справочник)
- HR и административный персонал (поддержка)

## 2. Функциональные требования

### FR-01: Главная страница

- Заголовок с приветствием и описанием портала
- Поле поиска по всей базе знаний (full-text по заголовкам, описаниям и содержимому статей)
- Плиточная сетка категорий с иконками, названиями, описаниями и счётчиками статей
- Адаптивная сетка: 3 колонки на десктопе, 2 на планшете, 1 на мобильном

### FR-02: Страница категории

- Заголовок категории с иконкой и описанием
- Нумерованный список статей с заголовками и описаниями
- Hover-эффекты для навигации
- Хлебные крошки (breadcrumbs)

### FR-03: Страница статьи

- Бейдж категории
- Заголовок статьи
- Полный текст статьи с поддержкой форматирования (pre-wrap)
- Поддержка внутренних ссылок между статьями (синтаксис `[[id|текст]]`)
- Блок «Ещё в этом разделе» — ссылки на другие статьи той же категории
- Хлебные крошки

### FR-04: Поиск

- Поиск в реальном времени (фильтрация при вводе)
- Поиск по полям: title, summary, content
- Результаты с бейджами категорий
- Стейт «ничего не найдено» с подсказкой обратиться к боту

### FR-05: AI-ассистент (чат-бот)

- Плавающая круглая кнопка (FAB) в правом нижнем углу
- Всплывающее окно чата поверх текущей страницы (370×500px)
- Сворачивание/разворачивание без потери истории
- Интеграция с Anthropic API (Claude Sonnet)
- Системный промпт ограничивает ответы строго базой знаний
- Контекст: все статьи передаются в каждый запрос
- Быстрые вопросы-подсказки в пустом чате
- Ввод API-ключа при первом обращении (хранение в state сессии)
- История последних 10 сообщений в контексте
- Индикатор загрузки (три пульсирующие точки)
- Анимация появления сообщений

### FR-06: Навигация

- Sticky-хедер с логотипом и кнопкой «Главная»
- Breadcrumbs на всех страницах кроме главной
- Scroll-to-top при переходе между страницами
- Кнопка «Назад» в хедере

## 3. Нефункциональные требования

### NFR-01: UI/UX

- Светлая тёплая тема (фон #faf9f7, акцент #e85d2a)
- Типографика: Fraunces (заголовки) + DM Sans (тело)
- CSS-переменные для всей темы
- Staggered animations при загрузке плиток
- Backdrop-filter blur на хедере
- Кастомные скроллбары
- Минимализм, плиточный интерфейс

### NFR-02: Производительность

- SPA без серверного рендеринга
- Lazy-загрузка не требуется (объём данных мал)
- Оптимизация: useMemo для поиска, useCallback для навигации

### NFR-03: Совместимость

- Chrome 90+, Firefox 90+, Safari 15+, Edge 90+
- Мобильные браузеры: iOS Safari, Chrome Android
- Минимальная ширина: 320px

### NFR-04: Безопасность

- API-ключ хранится только в React state (не в localStorage)
- API-ключ не логируется
- Заголовок `anthropic-dangerous-direct-browser-access` для клиентских вызовов
- MVP без авторизации; закрытие доступа — на уровне инфраструктуры (см. раздел 7)

```
├── data/

├── components/
│   ├── Layout.jsx              # Хедер + footer + обёртка
│   ├── Breadcrumb.jsx          # Хлебные крошки

│   ├── ArticleView.jsx         # Рендер статьи + связанные
│   ├── SearchBar.jsx           # Поисковая строка
│   ├── SearchResults.jsx       # Результаты поиска
│   └── ChatWidget/
│       ├── ChatWidget.jsx      # Контейнер виджета
│       ├── ChatFab.jsx         # FAB-кнопка
│       ├── ChatMessages.jsx    # Лента сообщений
│       └── ChatInput.jsx       # Поле ввода + отправка
├── hooks/
│   ├── useSearch.js            # Логика поиска
│   └── useChat.js              # Логика чата и API
├── styles/
│   └── global.css              # CSS-переменные, анимации, скроллбары
├── utils/
│   └── renderContent.js        # Парсер внутренних ссылок [[id|text]]
├── App.jsx                     # Корневой компонент, роутинг
└── main.jsx                    # Точка входа
```

> **Примечание:** Текущий прототип — single-file (`knowledge-portal.jsx`). Декомпозиция на компоненты выполняется в Milestone 2.

## 8. Стратегия развития

### Фаза 1 (MVP) — текущий скоуп

Всё описанное в данном ТЗ. Single-page React приложение, данные в коде, без авторизации.

### Фаза 2 — CMS

- Вынос данных во внешний JSON (отдельный файл в репозитории)
- Или интеграция с Google Sheets API как CMS для нетехнических редакторов
- Webhook для автообновления при изменении таблицы

### Фаза 3 — Закрытый доступ

- Cloudflare Access или Vercel Password Protection
- Или ограничение по VPN / IP

### Фаза 4 — Расширение

- Серверный API-прокси для Anthropic (скрыть ключ)
- Аналитика использования (популярные статьи, частые вопросы)
- Markdown-рендеринг в статьях
- Версионирование статей

## 9. Критерии приёмки

- Все 14 статей отображаются корректно
- Поиск находит статьи по любому слову из контента
- Чат-бот отвечает строго по базе знаний
- Чат открывается/сворачивается без потери истории
- Внутренние ссылки в статьях работают
- Breadcrumbs корректно отражают текущую позицию
- Портал корректно отображается на мобильных устройствах (320px+)
- Все тесты проходят (unit, component, e2e)
- Деплой на Vercel работает автоматически при push в main
